# 01-25
📦 数据持久化与存储
模块	能力	作用与实现要点
数据库设计	核心数据模型定义	设计 memos (笔记)、tags (标签)、resources (附件) 等表结构，并建立关联关系。
SQLite连接与初始化	在Go中初始化SQLite连接池，并自动执行建表、迁移脚本。
文件存储	本地文件系统管理	处理图片等附件上传，在指定目录（如 ./storage/）按日期/用户组织文件。
文件服务	通过Go静态文件服务或特定路由（如 /uploads/）提供附件访问。
🔌 API服务层
模块	能力	作用与实现要点
路由与控制器	RESTful API设计	实现 GET /api/memos, POST /api/memos, PUT /api/memos/:id, DELETE 等标准接口。
请求验证与绑定	使用Go的Struct Tag进行参数验证和JSON绑定，确保输入安全。
业务逻辑	核心CRUD服务	实现笔记的创建、读取（支持分页、过滤）、更新、删除逻辑。
标签系统	实现标签的创建、删除、合并，以及笔记与标签的多对多关联。
搜索与过滤	全文搜索	利用SQLite的FTS5扩展，为笔记内容建立全文搜索索引。
高级查询	实现按标签、日期范围、是否置顶、内容类型等进行组合过滤查询。
💻 前端交互层
模块	能力	作用与实现要点
页面与路由	单页面应用(SPA)	使用SvelteKit的客户端路由，提供流畅无刷新的页面切换体验。
响应式布局	确保应用在桌面、平板、手机上均有良好的显示效果。
编辑器	极简Markdown编辑器	实现一个支持基础Markdown语法（加粗、列表、链接）的即时渲染编辑器。
快速输入与保存	支持快捷键（如 Cmd/Ctrl + Enter）保存，输入框自动聚焦。
状态管理	客户端状态管理	使用Svelte Stores管理笔记列表、标签列表等全局状态，并保持与服务器同步。
离线支持	服务工作者(Service Worker)	注册Service Worker，缓存关键静态资源，提升加载速度并支持弱网环境。
本地草稿保存	在浏览器LocalStorage或IndexedDB中暂存未提交的笔记草稿，防止意外丢失。
🔐 用户认证与安全
模块	能力	作用与实现要点
访问控制	基础身份认证	实现简单的密码登录或Access Token认证（考虑到是私人应用，可简化）。
API请求鉴权	在所有受保护API路由的Go中间件中验证Token。
数据安全	SQL注入防护	使用Go的数据库驱动参数化查询，杜绝SQL注入。
输入净化与转义	对用户输入的内容进行适当的清理和转义，防止XSS攻击。
